# Short Portfolio Demonstration

Target: three to five minutes. Start paused with the server running and the prepare screen open.

1. State the problem: inspectable semantic chunks can reduce the navigation and rhythm costs of RSVP reading without surrendering policy to a parser provider.
2. Identify the checkout by the immutable SHA in the [handoff record](../validation/s043_portfolio_demonstration_rehearsal.md), then show `/health` and its configured/active chunking identity.
3. Paste [the prepared Markdown sample](../../samples/portfolio-demo.md), prepare it, and start then pause playback to show semantic chunk presentation and progress.
4. Move manually to a breakpoint or coarse-seek position; reload and show that restoration is paused.
5. Return to prepare, select [the project-owned EPUB](../../samples/portfolio-demo.epub), and show its `Opening` and `Continuation` contents entries. Select `Continuation` and confirm the reader stays paused.
6. Close with the project-owned `SourceDocument`/schedule boundaries, parser-assisted default with mandatory deterministic fallback, application-owned source-text non-persistence, and focused automated plus human validation.
7. State the principal limitation: EPUB support is deliberately narrow; ordinary books using `<link>`, `<meta>`, or a non-HTML5 doctype can be rejected explicitly.

If startup or browser presentation blocks the path, switch to the [bounded fallback](fallback_package.md); do not improvise with private or downloaded material.
